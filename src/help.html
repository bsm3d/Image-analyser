1. Overview

Image Analyzer is a privacy-focused web application designed to help detect whether an image has been generated by artificial intelligence (AI). All processing happens locally in your browser - no data is sent to any server.

Privacy First: This tool performs 100% client-side processing. Your images never leave your device. All analysis is performed directly in your web browser using JavaScript.

Key Features

  • AI Probability Scoring: Advanced algorithms analyze multiple image characteristics to estimate AI generation likelihood
  • EXIF Metadata Extraction: Reads camera and photo information embedded in image files
  • Visual Analysis Tools: Heatmaps, frequency analysis, and spectral simulations (UV/IR)
  • Training Mode: Improve detection accuracy with your own sample datasets
  • Multiple Export Formats: JSON, CSV, and TXT export options for further analysis
  • No Installation Required: Runs entirely in modern web browsers

What Makes This Tool Different

Unlike cloud-based AI detection services, Image Analyzer prioritizes user privacy by processing everything locally. This means faster results, no upload limits, and complete control over your data. The tool uses multiple detection methods including pattern recognition, texture analysis, color distribution, symmetry detection, and metadata validation.

2. Getting Started

Basic Analysis Workflow

  1. Click the Load image button in the main interface
  2. Select an image file from your device (JPEG, PNG, or WebP format)
  3. Wait for the analysis to complete (typically 2-5 seconds)
  4. Review the results displayed on screen

Analysis Components

The analysis provides comprehensive information across four main categories:

AI Probability Score

A percentage value (0-100%) indicating how likely the image is AI-generated. This score is calculated by analyzing multiple image characteristics and combining them into a weighted probability.

Detected Indicators

Specific features and patterns that suggest AI generation, including:

  • Repetitive patterns and textures
  • Sharp edge concentrations
  • Unnatural uniformity
  • Color palette characteristics
  • Symmetry anomalies
  • Digital noise signatures

EXIF Metadata

Camera and photo information extracted from the image file, if available. This includes camera model, lens information, exposure settings, GPS coordinates, timestamps, and software used.

Image Details

Technical information about the file including dimensions, file size, format, color space, and unique color count - a key indicator for AI detection.

Pro Tip: For best results, analyze the original, unedited image file. Heavy compression or editing can affect detection accuracy.

3. Understanding the Results

AI Probability Score Interpretation

The AI probability score is the primary indicator of whether an image is AI-generated. Understanding how to interpret this score is crucial for making informed decisions.

Score Range Interpretation Confidence Level
0-30% Likely a real photo High confidence - exhibits natural photographic characteristics
30-70% Uncertain - requires further analysis Medium confidence - mixed indicators present
70-100% Highly likely AI-generated High confidence - multiple AI indicators detected

Important Disclaimer: This tool provides probability-based analysis and should NOT be considered absolute proof. AI detection is an evolving field, and no method is 100% accurate. Use this tool as part of a broader verification process that includes human judgment and contextual analysis.

Detection Indicators Explained

The tool analyzes multiple aspects of the image to calculate the probability score:

Pattern Analysis

Examines repetitive patterns and sharp edges that are unnatural in real photographs. AI-generated images often exhibit perfectly repeated textures or unnaturally clean geometric patterns.

Texture Characteristics

Analyzes uniformity and gradient characteristics. Real photos typically have natural texture variation, while AI-generated images may show overly smooth or perfectly uniform textures.

Color Distribution

Evaluates color palette diversity and distribution. AI-generated images often have limited, carefully selected color palettes, while real photos exhibit more natural color variation and transitions.

Symmetry Detection

Identifies unusual symmetry patterns that may indicate AI generation. While some real-world objects are symmetrical, perfect symmetry across complex scenes is often a red flag.

Noise Characteristics

Distinguishes between natural camera sensor noise and artificial noise patterns. Real photos have characteristic noise patterns from the camera sensor, while AI-generated images may lack this or have artificial noise added.

Compression Artifacts

Analyzes compression artifacts and edge quality. The way an image compresses can reveal whether it originated from a camera or was generated digitally.

EXIF Metadata Significance

EXIF (Exchangeable Image File Format) data is one of the strongest pieces of evidence for determining image authenticity:

EXIF Status Interpretation Notes
Complete EXIF Present Strong indicator of real photo Camera settings, GPS, dates suggest genuine capture
Partial EXIF Present Possibly edited real photo Some editing software preserves partial metadata
EXIF Absent Common in AI images Also occurs in heavily edited or exported images
Suspicious EXIF Potentially fabricated Impossible settings or inconsistent data

Technical Note: Advanced users can potentially fake EXIF data using specialized tools. Therefore, EXIF presence should not be the only factor in your decision. Cross-reference with other indicators and visual analysis.

Color Count Analysis

The unique color count is a powerful indicator:

  • High Color Count (>10,000): Typical of real photographs with natural color variation
  • Medium Color Count (5,000-10,000): Could be compressed real photo or high-quality AI generation
  • Low Color Count (<5,000): Common in AI-generated images with limited color palettes

4. Visual Analysis Tools

Image Analyzer provides powerful visual overlays and transformations to help identify suspicious areas and characteristics. These tools simulate various imaging techniques used in forensic analysis.

Heatmap Visualization

The heatmap creates a color-coded overlay showing which areas of the image are suspicious based on local analysis of color diversity and edge characteristics. Click "Toggle Heatmap" to activate.

How the Heatmap Works

  • Divides the image into 16Ã—16 pixel blocks for localized analysis
  • Analyzes each block for:
    • Limited color palette (< 10 unique colors = suspicious)
    • Sharp edge concentration (many abrupt transitions)
    • Natural variation (> 50 colors = natural)
  • Assigns a suspicion score (0-100%) to each block
  • Applies color-coded overlay based on suspicion level

Color Code Interpretation

Color Suspicion Level Interpretation

Usage Tip: Hover over the heatmap to see the suspicion level for each area. Real photos typically show mostly green with occasional yellow spots. AI-generated images often have large red areas or unusual patterns of high suspicion zones.

Practical Examples

  • AI-Generated Portrait: Often shows red zones in background areas with perfect blur or texture, and in overly smooth skin areas
  • Real Portrait: Mostly green with yellow in genuinely blurred background, natural skin texture variation
  • AI-Generated Landscape: Red zones in unnaturally uniform sky, repetitive foliage textures, or perfect reflections
  • Real Landscape: Natural variation throughout, yellow in genuinely uniform areas like clear sky

Frequency Analysis

Displays the image's frequency spectrum, revealing patterns invisible to the naked eye. This technique is based on Fourier Transform analysis used in forensic image authentication. Click "Frequency" to activate.

How Frequency Analysis Works

  • Fast Fourier Transform (FFT): Converts spatial image data to frequency domain
  • Pattern Detection: Reveals periodic patterns, grid artifacts, and compression signatures
  • Artifact Identification: Shows JPEG compression blocks, upscaling artifacts, and processing traces

What to Look For

  • Regular Patterns: Grid-like patterns suggest AI generation or heavy processing
  • Compression Artifacts: Visible in real photos compressed as JPEG, but different signature in AI-generated images
  • Noise Distribution: Real camera sensor noise has characteristic frequency signatures
  • Upscaling Artifacts: Repeated patterns indicate image upscaling, common in AI generation

Expert Insight: Real photos show natural, non-uniform frequency distribution. AI-generated images often show artificial regularity or suspicious patterns in the frequency domain, especially in areas that appear smooth in the spatial domain.

Ultraviolet (UV) Mode

Simulates ultraviolet photography with false-color visualization. UV imaging is used in forensics to detect alterations and material properties. Click "Ultraviolet" to activate.

How UV Mode Works

  • UV channel simulation: Emphasizes blue/UV wavelengths (300-400nm)
  • False-color mapping: Blue = UV, Green = visible blue, Red = visible green
  • Material discrimination: Different materials reflect/absorb UV differently
  • Detail enhancement: Reveals hidden textures and surface properties

Characteristic UV Appearances

Material/Subject Real Photo UV Response AI Image Issues
Human Skin Appears darker, blemishes more visible Often shows unrealistic uniformity
Teeth/Eyes Teeth glow bright, eyes show UV fluorescence May lack realistic fluorescence
Flowers Show UV patterns invisible to human eye Usually lack accurate UV nectar guides
Fabrics Natural vs. synthetic show different absorption May not differentiate materials correctly
Sky Appears darker, increased contrast Often shows unrealistic sky rendering

AI Detection Applications

  • Skin Analysis: Real skin shows UV-absorbent properties; AI often fails to simulate this accurately
  • Material Consistency: Check if all elements in the scene behave according to UV physics
  • Fluorescence Effects: Real photos may show UV-induced fluorescence; AI typically doesn't simulate this
  • Detail Revelation: UV reveals skin texture, paint layers, and material age - often absent in AI images

Practical Examples

  • AI Portrait: Skin appears unrealistically uniform in UV, lacks natural blemishes and texture variation
  • Real Portrait: Skin shows increased texture, freckles, blemishes more visible, eyes and teeth may fluoresce
  • AI Flower: Lacks UV nectar guides that bees see (if the AI wasn't trained on UV photography)
  • Real Flower: Shows characteristic UV patterns that guide pollinators

Infrared (IR) Mode

Simulates near-infrared (NIR) photography with false-color visualization. IR imaging reveals heat signatures and material properties invisible in visible light. Click "Infrared" to activate.

How IR Mode Works

  • IR channel simulation: Enhances red/NIR wavelengths (700-900nm)
  • False-color mapping: Red = IR, Green = visible red, Blue = visible green
  • Vegetation index: NDVI-like calculation highlighting chlorophyll
  • Material discrimination: Different materials reflect IR differently

Characteristic IR Appearances

Material/Subject Real Photo IR Response AI Image Issues
Vegetation Appears bright magenta/red (highly IR-reflective) Often lacks characteristic IR plant response
Water Appears very dark (IR absorptive) May show unrealistic brightness
Human Skin Appears pale/translucent (IR penetration) Often overly smooth, lacks subcutaneous detail
Clouds Remain white/bright (scatter all wavelengths) Usually handled correctly by AI
Synthetic Materials Show distinctive IR signatures May not differentiate materials accurately

AI Detection Applications

  • Physical Accuracy: AI may not simulate correct IR material properties consistently
  • Vegetation Detection: Real plants show strong IR response due to chlorophyll; AI-generated foliage often fails this test
  • Atmospheric Effects: Real IR shows haze penetration; AI often fails to simulate this accurately
  • Skin Translucency: Real skin in IR shows subcutaneous detail; AI typically produces overly smooth results
  • Material Consistency: Check if all elements show physically accurate IR behavior throughout the scene

Practical Examples

  • AI Landscape: Trees may not show characteristic IR magenta/red coloration, or show it inconsistently
  • Real Landscape Photo: Grass and foliage appear bright magenta due to chlorophyll IR reflectance
  • AI Portrait: Skin may lack realistic IR translucency and subcutaneous vein visibility
  • Real Portrait: Skin shows IR penetration, freckles/blemishes less visible, veins more visible
  • AI Sky: Lacks realistic atmospheric IR scattering, haze appears too dense
  • Real Sky: Shows haze penetration, distant objects more visible than in visible light

Pro Tip: Use UV and IR modes together! Real photos should show physically consistent behavior across UV and IR spectra. AI-generated images often fail one or both spectral tests. If an image passes both UV and IR consistency checks, it's much more likely to be a real photograph.

Using Multiple Analysis Modes Together

The most effective analysis strategy combines multiple visualization modes:

  1. Start with Heatmap: Quickly identify suspicious areas
  2. Apply Frequency Analysis: Check for artificial patterns and processing artifacts
  3. Test with UV Mode: Verify material behavior and detail consistency
  4. Validate with IR Mode: Confirm physical accuracy and chlorophyll response
  5. Cross-Reference: Areas flagged by multiple modes are most suspicious

5. Analysis Modes, Export Options & Training

Analysis Modes

Image Analyzer offers two distinct analysis modes, each with different detection approaches. Use the Analysis Mode Selector dropdown in the main interface to switch between them.

Heuristic Analysis Mode (Default)

Uses fixed, recalibrated thresholds based on statistical analysis of known AI and real images. This mode:

  • Fixed Thresholds: Uses predetermined detection thresholds optimized for general use
  • Fast Analysis: Immediate results without requiring training data
  • Consistent Results: Same thresholds applied to all images
  • Best For: General purpose analysis, quick checks, unknown image sources
  • Detection Criteria: Pattern repetition, texture uniformity, color palette, noise characteristics

AI Trainable Analysis Mode

Uses adaptive thresholds that can be customized through the Training Mode. This mode:

  • Adaptive Thresholds: Detection thresholds adjust based on your training samples
  • Customizable: Fine-tune detection for specific AI generators or image types
  • Learning Capability: Improves accuracy when trained with verified samples
  • Best For: Specialized analysis, specific AI generators (Midjourney, DALL-E, etc.), domain-specific detection
  • Requires Training: Most effective after training with known AI and real image samples

Which Mode Should You Use?

Situation Recommended Mode Reason
General image verification Heuristic No training required, reliable for most cases
Unknown image sources Heuristic Broad detection coverage
Specific AI generator (Midjourney, DALL-E) AI Trainable (after training) Can be optimized for generator's characteristics
Batch analysis of similar images AI Trainable (after training) Learns patterns specific to your dataset
First-time use Heuristic Works immediately without setup

Pro Tip: When switching to AI Trainable mode, the Training Mode button becomes visible. Use it to train the detector with your verified AI and real image samples for improved accuracy.

Export Functionality

Image Analyzer allows you to export analysis results in multiple formats for documentation, further analysis, or integration with other tools.

Available Export Formats

JSON Export

Exports complete analysis data in JSON (JavaScript Object Notation) format, ideal for programmatic use, data analysis, or integration with other applications. The JSON file includes:

  • AI probability score and all component scores
  • Detected indicators with confidence levels
  • Complete EXIF metadata
  • Image technical specifications
  • Color analysis data
  • Timestamp of analysis

CSV Export

Exports key metrics in CSV (Comma-Separated Values) format, compatible with spreadsheet applications like Microsoft Excel, Google Sheets, or LibreOffice Calc. Perfect for:

  • Batch analysis comparison
  • Statistical analysis
  • Creating charts and graphs
  • Database import

TXT Export

Exports EXIF data in human-readable text format. This format is ideal for:

  • Quick reference and documentation
  • Including in reports or presentations
  • Printing or email sharing
  • Easy readability without special software

Training Mode

Training Mode is an advanced feature that allows you to improve detection accuracy by providing sample images of known AI-generated and real photos. The system analyzes your samples and adjusts detection thresholds accordingly.

How to Use Training Mode

  1. Click the Training Mode button in the main interface
  2. In the "AI-Generated Images" section, load at least 5 images you know are AI-generated
  3. In the "Real Images" section, load at least 5 images you know are real photographs
  4. Click Start Training to begin the analysis
  5. Wait for the training process to complete (may take 1-2 minutes)
  6. Review the training report showing characteristic differences found

Training Tips: The more diverse your training samples, the better the detector will perform. Include various subjects, styles, and lighting conditions in your training set. Maximum 100 images per type to prevent memory issues.

What Training Does

  • Feature Analysis: Examines characteristics of both AI-generated and real image sets
  • Statistical Comparison: Calculates statistical differences between the two groups
  • Threshold Adjustment: Automatically adjusts detection thresholds based on your samples
  • Accuracy Improvement: Enhances future detection accuracy for similar image types
  • Report Generation: Provides detailed analysis of discovered patterns

Training Report Interpretation

After training, you'll receive a report showing:

  • Average AI probability scores for each group
  • Key differentiating features discovered
  • Color palette differences
  • Texture and pattern variations
  • EXIF metadata presence statistics
  • Recommended detection threshold adjustments

Important: Training is most effective when you're certain about the origin of your sample images. Including mislabeled images will decrease accuracy rather than improve it. Quality of training data is more important than quantity.

When to Use Training Mode

  • When working with images from specific AI generators (Midjourney, DALL-E, Stable Diffusion, etc.)
  • When analyzing a specific type of content (portraits, landscapes, abstract art)
  • When the default detection seems less accurate for your use case
  • When building a specialized detection system for a particular domain

6. Best Practices & Limitations

Best Practices for Accurate Detection

  1. Use Multiple Detection Methods: Don't rely solely on this tool or any single indicator. Combine automated detection with visual inspection, contextual analysis, and other verification methods.
  2. Analyze Original Files: Whenever possible, analyze the original, unedited image file. Compression, resizing, or editing can alter detection characteristics.
  3. Consider the Context: Where did the image come from? Who provided it? What is its claimed purpose? Context can provide crucial clues.
  4. Look for Logical Inconsistencies: Check for anatomical impossibilities, incorrect physics (lighting, shadows, reflections), or objects that don't make sense in the scene.
  5. Verify EXIF Data Carefully: If EXIF is present, verify that camera model and settings make sense. Check for impossible combinations (e.g., ISO 100 with 1/8000s shutter in darkness).
  6. Use Training Mode: If you regularly analyze similar types of images, use Training Mode with verified samples to improve accuracy.
  7. Stay Updated: AI technology evolves rapidly. What works today may be less effective tomorrow. Keep learning about new AI capabilities and detection methods.
  8. Keep Expectations Realistic: No tool can guarantee perfect detection. Accept that some images will be difficult or impossible to classify with certainty.

Known Limitations

Critical Limitations to Understand:

  • No 100% Accuracy: No AI detection method is perfect. Both false positives and false negatives occur.
  • Evolving AI Technology: AI image generation improves rapidly. New models may bypass current detection methods.
  • Image Quality Dependence: Results depend heavily on image quality, format, and compression level.
  • Edited Real Photos: Heavily edited or filtered real photographs may incorrectly appear as AI-generated.
  • High-Quality AI Images: Some AI-generated images are virtually indistinguishable from real photos.
  • Hybrid Content: Images that combine real photos with AI-generated elements are particularly challenging.
  • Processing Limitations: Client-side processing has memory and performance constraints compared to cloud-based solutions.
  • Format Limitations: Some image formats or color spaces may not be fully supported.

False Positives and False Negatives

Common Causes of False Positives (Real Photos Flagged as AI)

  • Heavy post-processing, filters, or beauty mode effects
  • Professional photo retouching and compositing
  • Certain smartphone camera processing algorithms
  • Miniature photography or tilt-shift effects
  • Studio photography with perfect lighting and backgrounds
  • Images with removed or stripped EXIF data

Common Causes of False Negatives (AI Images Not Detected)

  • Latest-generation AI models trained to avoid detection
  • AI images with added fake EXIF data
  • AI images post-processed to add realistic imperfections
  • High-quality AI generation with photorealistic rendering
  • AI images based on real photo references or datasets

Ethical Considerations

Responsible Use: This tool should be used responsibly and ethically. Consider the following:

  • Respect people's privacy when analyzing images containing individuals
  • Don't use results to unfairly accuse or defame others without additional evidence
  • Understand that AI-generated content isn't inherently bad or deceptive
  • Consider the legal implications in your jurisdiction before making public claims
  • Be transparent about using automated detection tools and their limitations

Future-Proofing Your Detection Strategy

As AI technology advances, maintain effectiveness by:

  • Regularly updating your knowledge of new AI models and capabilities
  • Combining multiple detection tools and methods
  • Focusing on fundamental physical and logical inconsistencies that AI struggles with
  • Building expertise in human visual analysis alongside automated tools
  • Participating in communities focused on AI detection and digital forensics

System Requirements

Browser Compatibility

Hardware Requirements

  • RAM: Minimum 4GB, 8GB+ recommended for large images
  • Processor: Modern multi-core processor (Intel i5/AMD Ryzen 5 or better)
  • Storage: At least 100MB free space for browser cache
  • Display: Minimum 1366x768 resolution, 1920x1080 or higher recommended

Software Requirements

  • JavaScript: Must be enabled in browser settings
  • Cookies: Session cookies required for proper functionality
  • Canvas API: Required for image processing (supported by all modern browsers)
  • File API: Required for local file access

Supported Image Formats

Format Extension Notes
JPEG .jpg, .jpeg Most common format, full support with EXIF
PNG .png Lossless format, full support, limited EXIF
WebP .webp Modern format, efficient compression, full support

Image Limitations

  • Maximum Dimensions: 4096 x 4096 pixels
  • Maximum File Size: 15MB
  • Color Space: RGB (sRGB recommended)
  • Bit Depth: 8-bit per channel (24-bit RGB)

Troubleshooting Common Issues

Image Won't Load

Symptoms: Error message when selecting image, or no response after file selection

Possible Causes & Solutions:

  • Unsupported Format: Ensure file is JPEG, PNG, or WebP. Convert if necessary.
  • File Too Large: Check file size is under 15MB. Reduce image size if needed.
  • Image Too Large: Verify dimensions are within 4096 x 4096 pixel limit. Resize if needed.
  • Corrupted File: Try opening the image in another program to verify it's not corrupted.
  • Browser Issues: Clear browser cache, try a different browser, or restart your browser.

Analysis is Slow or Freezing

Symptoms: Analysis takes more than 30 seconds, browser becomes unresponsive

Possible Causes & Solutions:

  • Large Image File: Large, high-resolution images take longer. Consider reducing size to 2048px maximum dimension.
  • Low Memory: Close other browser tabs and applications to free up RAM.
  • Browser Performance: Try using Chrome or Edge which typically have better performance for heavy JavaScript.
  • Multiple Analyses: Wait for current analysis to complete before loading another image.
  • System Resources: Check if other programs are using significant CPU/RAM and close them.

Heatmap or Visual Tools Not Working

Symptoms: Clicking visualization buttons has no effect

Possible Causes & Solutions:

  • Image Not Loaded: Ensure you've successfully loaded and analyzed an image first.
  • Canvas Not Supported: Update your browser to a version that supports HTML5 Canvas API.
  • Memory Issues: Close other tabs and refresh the page to free up memory.
  • JavaScript Errors: Open browser console (F12) to check for errors. Report any errors found.

Training Mode Fails or Crashes

Symptoms: Training doesn't start, browser crashes, or error message appears

Possible Causes & Solutions:

  • Insufficient Images: Ensure you have at least 5 images of each type (AI and real).
  • Corrupted Images: Verify all training images are valid and not corrupted.
  • Too Many Images: Try with fewer images if experiencing memory issues (start with 10-20 total).
  • Memory Exhaustion: Close all other applications and browser tabs before training.
  • Mixed Image Sizes: Very large images in training set can cause issues. Resize to maximum 1024px.

Export Functions Not Working

Symptoms: Export buttons don't respond or downloads fail

Possible Causes & Solutions:

  • No Analysis Data: Ensure you've analyzed an image before attempting export.
  • Browser Download Settings: Check if browser is blocking downloads or has download restrictions.
  • Popup Blocker: Disable popup blocker for this site if it's interfering with downloads.
  • Storage Permission: Grant storage/download permissions when browser prompts.

Inaccurate Results

Symptoms: Detection results don't match expectations

Possible Causes & Solutions:

  • Heavily Edited Image: Remember that edited real photos may score high on AI probability.
  • Low-Quality Image: Poor quality or heavily compressed images are harder to analyze accurately.
  • Advanced AI Model: Latest AI models may bypass detection. Use multiple verification methods.
  • Use Training Mode: Train the detector with samples specific to your use case.
  • Check All Indicators: Don't rely solely on the probability score. Review all detection indicators.

Performance Optimization Tips

  • Image Preprocessing: Resize very large images to 2048px maximum dimension before analysis
  • Browser Selection: Use Chrome or Edge for best performance with heavy computation
  • System Optimization: Close unnecessary applications to free up RAM
  • Cache Management: Clear browser cache periodically if experiencing issues
  • Extension Conflicts: Disable browser extensions if they might interfere with canvas operations
